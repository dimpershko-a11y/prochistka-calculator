# PRO-CHISTKA калькулятор v4.7.0

Мини-приложение собрано из рабочего файла `calculator-calculator-1111.html`.

## Версионирование

Версия имеет вид `MAJOR.MINOR.PATCH` (например, `4.5.3`):

- **PATCH** (третье число) — мелкие правки: тексты, отдельные строки в смете, мелкие стили.
- **MINOR** (второе число) — заметные изменения: новые функции и блоки, переработка логики.
- **MAJOR** (первое число) — глубокие/крупные изменения структуры приложения.

Версия указывается в `config.js` (`APP_VERSION`), дублируется в бейдже на странице и в имени кэша Service Worker (`sw.js`) — при выпуске новой версии меняйте её во всех трёх местах.

## Главное

- `index.html` — программа калькулятора.
- `assets/app.js` — логика программы. При обновлениях меняется этот файл.
- `assets/style.css` — стили.
- `config.js` — все настраиваемые данные: ставки, коэффициенты, доп. услуги, описания услуг, способы оплаты, бренд, пароль.
- `tilda_iframe_example.html` — пример iframe-кода для Tilda.

## Что вынесено в config.js

- базовые ставки и минималки;
- скорости работ;
- коэффициенты заставленности и загрязнённости;
- список доп. услуг, цены, категории, единицы измерения и время;
- описания “что входит” для каждого вида уборки;
- отдельное описание работ по окнам / остеклению;
- способы оплаты;
- данные компании и PDF-настройки;
- пароль доступа к изменениям.

## Описание окон

Текст `defaults.serviceDescriptions.windows` добавляется в смету и PDF только если в заказе есть выбранные услуги из категории `Окна`.

## Пароль

Пароль теперь не закрывает весь калькулятор. Без пароля можно считать, выбирать услуги, сохранять заказы и формировать смету. Пароль нужен только для изменения тарифов, настроек, доп. услуг, описаний и восстановления резервной копии.

Пароль работает в браузере и защищает настройки от случайного изменения. Он не является серверной авторизацией и не должен использоваться для хранения секретных данных.

Пароль задаётся в `config.js`:

```js
APP_PASSWORD: '7421'
```

Чтобы отключить пароль, оставьте пустую строку:

```js
APP_PASSWORD: ''
```

## Обновления

При обновлении программы старайтесь заменять только:

- `index.html`
- `assets/app.js`
- `assets/style.css`
- `manifest.json`
- `sw.js`

Файл `config.js` не трогайте, чтобы не потерять цены, описания и настройки.

### 4.6.1 — патч
Обновлены рыночные ставки и минималки (свериться с рынком): генеральная 300 ₽/м² (мин 12 000), после ремонта 390 ₽/м² (мин 18 000), поддерживающая 150 ₽/м² (мин 6 000). Поднята `CONFIG_REVISION` (2 → 3), чтобы новые цены подтянулись из конфига на всех устройствах поверх прежних сохранённых.

### 4.6.0 — переработка расчёта себестоимости (значимый релиз)
Полностью переосмыслена экономика расчёта. Раньше себестоимость считалась как «фикс × число сотрудников» с наценкой только на ФОТ; накладные, материалы и труд владельца не учитывались.

Теперь модель строится снизу вверх:
- **Труд по дневным ставкам и роли на объекте.** Число людей выводится автоматически из нормо-часов и лимита часов на человека в день (чтобы уложиться в один рабочий день). Роль владельца на объекте: «Не еду» / «Менеджер» (контроль, +день сверх бригады) / «Клинер+менеджер» (занимает место в бригаде по своей ставке). Труд владельца теперь оплачивается внутри себестоимости.
- **Материалы** — отдельной строкой по ₽/м².
- **Накладные на заказ** — постоянные расходы в месяц делятся на число заказов в месяц. Снижается автоматически при росте числа заказов.
- **Жёсткий пол** (труд + материалы) — ниже него заказ убыточен. **Полная себестоимость** (+ накладные). **Целевая цена** (+ наценка прибыли, по умолчанию 25%). Рекомендация = максимум(рынок, целевая), но не ниже жёсткого пола.
- Сводка показывает рынок, жёсткий пол, полную себестоимость, целевую цену, фактическую прибыль и маржу. Предупреждение срабатывает, когда рынок ниже полной себестоимости (или, отдельно, ниже прямых затрат).
- Ставки труда, материалы и накладные редактируются в разделе «Тарифы».

Параметры по умолчанию: клинер 5000 ₽/день, владелец-менеджер 5000, клинер+менеджер 7000, лимит 9 ч/день, материалы 15 ₽/м², накладные 75 000 ₽/мес при 10 заказах, целевая прибыль 25%. Изменения формы: блок «Сотрудники и прибыль» заменён на «Бригада и прибыль» (роль на объекте + целевая прибыль); поля «Кол-во сотрудников», «Расчёт оплаты», «ЗП сотрудника» убраны (число людей считается автоматически). При первом запуске старый процент прибыли сбрасывается на 25%, так как его смысл изменился.

### 4.5.0 — значимый релиз
Объединяет работу по данным, скидке, выезду и визуалу (разделы ниже).

### Изменение цен и ревизия конфига (важно)
Раньше правка `config.js` не меняла цены на устройстве, где приложение уже использовалось (локальные данные перекрывали конфиг). Теперь это исправлено через ревизию.

- Цены, коэффициенты и выезд задаются в `config.js`.
- Чтобы изменения применились на всех устройствах, после правки цен **увеличьте `CONFIG_REVISION`** (например, с `1` на `2`).
- При следующем открытии приложение увидит новую ревизию и перезапишет ставки/коэффициенты/выезд из `config.js`.
- Быстрые правки на месте по-прежнему можно делать в разделе «Тарифы»; они действуют до следующего повышения `CONFIG_REVISION`.

```js
"CONFIG_REVISION": 3,
```

### Выезд вынесен в конфиг
Стоимость выезда больше не зашита в коде. Блок `defaults.travel`:

```js
"travel": {
  "kad":      { "label": "В пределах КАД",  "base": 0,    "perKm": 0  },
  "km15":     { "label": "До 15 км от КАД", "base": 1500, "perKm": 0  },
  "km20plus": { "label": "20+ км",          "base": 1500, "perKm": 15 }
}
```
`base` — фиксированная стоимость, `perKm` — доплата за километр (умножается на введённые километры). Поле «Километры» показывается только когда `perKm > 0`.

### Скидка
- Скидка считается **от базовой стоимости уборки и доп. услуг**. Стоимость выезда не дисконтируется.
- Появился выбор режима: скидка в процентах (`%`) или фиксированной суммой (`₽`). Скидка суммой не может превышать базу + доп. услуги.

### PDF и текстовая смета без служебных предупреждений
Предупреждение «цена ниже экономики» теперь показывается только в самом калькуляторе (оператору). В сохранённый PDF и в копируемую смету оно больше не попадает.

### Защита от потери данных
- При запуске приложение запрашивает у браузера постоянное хранилище (`navigator.storage.persist`), чтобы данные реже вытеснялись.
- Состояние дублируется в IndexedDB. Если `localStorage` окажется пустым, данные автоматически восстанавливаются из резервного хранилища.
- Карточка «Резервное копирование» показывает дату последней копии и число новых заказов, напоминает сделать копию и умеет включать авто-скачивание копии каждые 10 заказов.
- Важное: всё это снижает риск, но не заменяет привычку периодически скачивать `Резервную копию` и хранить файл вне браузера (облако/флешка).

### Визуал
- Более читабельный системный шрифт, увеличен базовый размер текста, смягчены сверхжирные начертания, повышен контраст подписей.
- Карточки доп. услуг переработаны: название отдельной строкой сверху, поля «Кол-во / Цена / Время» выровнены в ровную сетку.
